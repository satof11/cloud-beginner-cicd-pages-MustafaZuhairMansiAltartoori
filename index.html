What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Delivery/Deployment. Continuous Integration means pushing changes frequently and running automated checks for quick feedback. Continuous Delivery/Deployment means verified changes can be released automatically to a live environment. In cloud systems, this reduces manual work, speeds up releases, and improves reliability.

My Pipeline

When I push code to the main branch, GitHub Actions starts a workflow run automatically. It checks that required files exist, uploads the website as an artifact, then deploys it to GitHub Pages. After the run succeeds, the live site updates with my latest commit.

  • Push → workflow starts
  • Checks → required files verified
  • Deploy → Pages updated
